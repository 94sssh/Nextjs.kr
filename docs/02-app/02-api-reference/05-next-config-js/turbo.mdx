> **주의**: 해당 내용은 실험적인 기능이며 `next --turbo`로만 동작합니다.

## webpack 로더

현재, Turbopack은 webpack 로더 API의 일부분을 지원하기 때문에 몇 가지 webpack 로더를 사용하여 Turbopack 코드를 변환할 수 있습니다.

로더를 구성하기 위해서, 설치한 로더의 이름과 `next.config.js`옵션을 추가하여 파일 확장자로 로더 목록에 매핑하세요 :

```js filename="next.config.js"
module.exports = {
  experimental: {
    turbo: {
      loaders: {
        // 옵션 형식
        '.md': [
          {
            loader: '@mdx-js/loader',
            options: {
              format: 'md',
            },
          },
        ],
        // 옵션이 없는 형식
        '.mdx': ['@mdx-js/loader'],
      },
    },
  },
}
```

이후 주어진 위 구성을 적용하여 앱에서 변환된 코드를 사용할 수 있습니다:

```js
import MyDoc from './my-doc.mdx'

export default function Home() {
  return <MyDoc />
}
```

## 분석 별칭

Turbopack은 `next.config.js`을 통해 webpack의 [`resolve.alias`](https://webpack.js.org/configuration/resolve/#resolvealias)구성과 유사하게 모듈 분석 처리를 별칭으로 변경할 수 있습니다.

별칭을 구성하려면 모듈 가져오기 패턴을 `next.config.js`의 새로운 대상에 매핑하세요:

```js filename="next.config.js"
module.exports = {
  experimental: {
    turbo: {
      resolveAlias: {
        underscore: 'lodash',
        mocha: { browser: 'mocha/browser-entry.js' },
      },
    },
  },
}
```

이 별칭은 `underscore` 패키지를 `lodash`로 가져옵니다. 쉽게 말해 `import underscore from 'underscore'`는 `underscore`대신 `lodash`를 로드합니다.

또한 Turbopack은 Node.js의 [conditional exports](https://nodejs.org/docs/latest-v18.x/api/packages.html#conditional-exports)과 유사하게 해당 필드를 통해 조건부 별칭을 지원합니다. 현재는 `browser`조건만 지원합니다. 위의 경우 Turbopack이 브라우저 환경을 대상으로 할 때 `mocha`모듈 가져오기는 `mocha/browser-entry.js`로 별칭처리됩니다.

더 많은 정보와 webpack에서 Turbopack으로 app을 변경하는 가이드를 알고 싶다면, [Turbopack's documentation on webpack compatibility](https://turbo.build/pack/docs/migrating-from-webpack)를 참조하세요.
