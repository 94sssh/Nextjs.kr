---

도메인 하위 경로에 Next.js 어플리케이션을 배포하려면 `basePath` 옵션을 사용하면 됩니다. 

`basePath`는 어플리케이션에 경로 접두사를 설정할 수 있습니다. 예컨데 `/docs`를 `''`대신 사용하고 싶은 경우(빈 문자열이 기본값), `next.config.js` 파일을 열어 `basePath` 옵션을 추가합니다:

```js filename="next.config.js"
module.exports = {
  basePath: '/docs',
}
```

> **알아두면 좋은 것**: 이 값의 경우 반드시 빌드 할 때 설정해야하며, 해당 값이 클라이언트 사이드 번들에 들어가서 다시 빌딩하지 않는 이상 바뀌지 않습니다.

### Links

`next/link`와 `next/router`를 이용해서 다른 페이지들을 연결하는 경우, `basePath`는 자동으로 적용됩니다. 

예컨데 `basePath`가 `/docs`로 설정된 경우 `/about`은 자동적으로 `/docs/about`가 됩니다.

```js
export default function HomePage() {
  return (
    <>
      <Link href="/about">About Page</Link>
    </>
  )
}
```

결과물:

```html
<a href="/docs/about">About Page</a>
```

이는 `basePath` 값을 변경할 때 어플리케이션의 모든 링크를 변경하지 않아도 된다는 의미입니다.

### Images

[`next/image`](/docs/pages/api-reference/components/image) 컴포넌트를 사용할 경우, `basePath`를 `src` 경로 앞에 붙여야합니다.

예컨데 `basePath`가 `/docs`인 경우, `/docs/me.png`를 사용하면 이미지가 잘 나옵니다.

```jsx
import Image from 'next/image'

function Home() {
  return (
    <>
      <h1>My Homepage</h1>
      <Image
        src="/docs/me.png"
        alt="Picture of the author"
        width={500}
        height={500}
      />
      <p>Welcome to my homepage!</p>
    </>
  )
}

export default Home
```
