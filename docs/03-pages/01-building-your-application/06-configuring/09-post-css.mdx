## 기본 동작

Next.js는 [내장된 CSS 지원](/docs/pages/building-your-application/styling)을 위해 PostCSS를 사용하여 CSS를 컴파일합니다.

Next.js는 다음과 같이 별도의 구성 없이 CSS를 변환하여 컴파일합니다.

- [Autoprefixer](https://github.com/postcss/autoprefixer) 는 자동으로 CSS 규칙에 접두사(vendor prefixes)를 추가합니다. (IE11 호환)
- [Cross-browser Flexbox bugs](https://github.com/philipwalton/flexbugs) 는 [사양](https://www.w3.org/TR/css-flexbox-1/)에 맞게 동작하도록 수정되었습니다.
- Internet Explorer 11과의 호환성을 위해 새로운 CSS 기능들이 자동으로 컴파일 됩니다:
  - [`all` 속성](https://developer.mozilla.org/en-US/docs/Web/CSS/all)
  - [Break 속성들](https://developer.mozilla.org/en-US/docs/Web/CSS/break-after)
  - [`font-variant` 속성](https://developer.mozilla.org/en-US/docs/Web/CSS/font-variant)
  - [Gap 속성들](https://developer.mozilla.org/en-US/docs/Web/CSS/gap)
  - [미디어 쿼리 범위](https://developer.mozilla.org/en-US/docs/Web/CSS/Media_Queries/Using_media_queries#Syntax_improvements_in_Level_4)

기본적으로 IE11 지원을 위해 [CSS 그리드](https://www.w3.org/TR/css-grid-1/)와 [커스텀 속성들](https://developer.mozilla.org/en-US/docs/Web/CSS/var)(CSS 변수들)은 **컴파일하지 않습니다.**

IE11용 [CSS 그리드 레이아웃](https://developer.mozilla.org/en-US/docs/Web/CSS/grid)을 컴파일하려면 CSS 파일 상단에 다음 주석을 추가할 수 있습니다.
```css
/* autoprefixer grid: autoplace */
```

또한 아래와 같이 config 파일에 autoprefixer를 추가하여 전체 프로젝트에 IE11에서의 [CSS 그리드 레이아웃](https://developer.mozilla.org/en-US/docs/Web/CSS/grid) 지원을 활성화 할 수 있습니다.(접힌 글)
더 자세한 정보는 ["플러그인 커스터마이징"](#customizing-plugins)에서 확인해주세요.

<details>
  <summary>CSS 그리드 레이아웃을 지원하기 위한 config 파일 설정을 보려면 클릭하세요.</summary>

```json filename="postcss.config.json"
{
  "plugins": [
    "postcss-flexbugs-fixes",
    [
      "postcss-preset-env",
      {
        "autoprefixer": {
          "flexbox": "no-2009",
          "grid": "autoplace"
        },
        "stage": 3,
        "features": {
          "custom-properties": false
        }
      }
    ]
  ]
}
```

</details>


CSS 변수는 [안전하게 컴파일할 수 없기 때문에](https://github.com/MadLittleMods/postcss-css-variables#caveats) 컴파일되지 않습니다.
변수를 꼭 사용해야 한다면 [Sass](https://sass-lang.com/)로 컴파일 되는 [Sass 변수](https://sass-lang.com/documentation/variables)를 고려해보세요.

## 브라우저 대상 커스터마이징

Next.js에서 [Browserslist](https://github.com/browserslist/browserslist)를 통해 대상 브라우저에 대해 설정할 수 있습니다.([Autoprefixer](https://github.com/postcss/autoprefixer)와 컴파일된 CSS 기능들을 위함)

 browserslist를 설정하려면 다음과 같이 `package.json` 파일에  `browserslist` 키 값을 생성하세요:

```json filename="package.json"
{
  "browserslist": [">0.3%", "not dead", "not op_mini all"]
}
```

[browsersl.ist](https://browsersl.ist/?q=%3E0.3%25%2C+not+ie+11%2C+not+dead%2C+not+op_mini+all) 도구를 사용하면 대상 브라우저를 시각화 할 수도 있습니다.

## CSS 모듈

No configuration is needed to support CSS Modules. To enable CSS Modules for a file, rename the file to have the extension `.module.css`.
CSS 모듈 지원을 위한 설정이 따로 필요하지 않습니다. 파일에서 CSS 모듈을 사용하고 싶다면 파일 이름을 확장자 `.module.css`로 변경합니다.

You can learn more about [Next.js' CSS Module support here](/docs/pages/building-your-application/styling).

## Customizing Plugins

> **Warning**: When you define a custom PostCSS configuration file, Next.js **completely disables** the [default behavior](#default-behavior).
> Be sure to manually configure all the features you need compiled, including [Autoprefixer](https://github.com/postcss/autoprefixer).
> You also need to install any plugins included in your custom configuration manually, i.e. `npm install postcss-flexbugs-fixes postcss-preset-env`.

To customize the PostCSS configuration, create a `postcss.config.json` file in the root of your project.

This is the default configuration used by Next.js:

```json filename="postcss.config.json"
{
  "plugins": [
    "postcss-flexbugs-fixes",
    [
      "postcss-preset-env",
      {
        "autoprefixer": {
          "flexbox": "no-2009"
        },
        "stage": 3,
        "features": {
          "custom-properties": false
        }
      }
    ]
  ]
}
```

> **Good to know**: Next.js also allows the file to be named `.postcssrc.json`, or, to be read from the `postcss` key in `package.json`.

It is also possible to configure PostCSS with a `postcss.config.js` file, which is useful when you want to conditionally include plugins based on environment:

```js filename="postcss.config.js"
module.exports = {
  plugins:
    process.env.NODE_ENV === 'production'
      ? [
          'postcss-flexbugs-fixes',
          [
            'postcss-preset-env',
            {
              autoprefixer: {
                flexbox: 'no-2009',
              },
              stage: 3,
              features: {
                'custom-properties': false,
              },
            },
          ],
        ]
      : [
          // No transformations in development
        ],
}
```

> **Good to know**: Next.js also allows the file to be named `.postcssrc.js`.

Do **not use `require()`** to import the PostCSS Plugins. Plugins must be provided as strings.

> **Good to know**: If your `postcss.config.js` needs to support other non-Next.js tools in the same project, you must use the interoperable object-based format instead:
>
> ```js
> module.exports = {
>   plugins: {
>     'postcss-flexbugs-fixes': {},
>     'postcss-preset-env': {
>       autoprefixer: {
>         flexbox: 'no-2009',
>       },
>       stage: 3,
>       features: {
>         'custom-properties': false,
>       },
>     },
>   },
> }
> ```
